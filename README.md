# 台灣手語即時辨識 AI 系統

即時辨識台灣手語詞彙，透過瀏覽器攝影機擷取畫面，以 MediaPipe 提取手部與身體特徵，送入 BiLSTM 模型分類，結果即時顯示於網頁。

> 自主學習計畫作品：從閱讀 *Attention Is All You Need* 論文、手推 Transformer 公式，到將 Attention 機制應用於手語序列辨識模型，3 天內完成可公開使用的 AI 網站。

[![Built with Claude Code](https://img.shields.io/badge/Built%20with-Claude%20Code-black?logo=anthropic)](https://claude.ai/code)

---

## 功能

- 瀏覽器攝影機即時辨識，無需安裝任何軟體
- 支援 32 個台灣手語常用詞彙
- Top-3 候選詞彙即時顯示，附信心度進度條
- 手停止時自動觸發 EndPose Hybrid 辨識（提升靜態手勢準確率）
- 詞彙累積記錄，形成句子
- 頁面內建詞彙字典，告知使用者可辨識範圍

## 系統架構

```
瀏覽器攝影機（每 100ms 截圖）
    ↓  WebSocket
Flask 後端（app.py）
    ↓
MediaPipe HandLandmarker + PoseLandmarker
→ 179 維特徵向量（雙手關鍵點 + 身體錨點 + 手臂 + 手臉距離 + 身體朝向 + 手指伸展）
    ↓
狀態機（IDLE → COLLECTING → 觸發）
    ↓
BiLSTM + Attention Pooling
→ 32 類詞彙 Top-3
    ↓  WebSocket
瀏覽器即時顯示
```

## 從 Transformer 理論到實際應用

本專案的核心模型使用 **BiLSTM + Attention Pooling**。Attention Pooling 直接源自 Transformer 的 Scaled Dot-Product Attention 機制：對序列中每個時間步計算重要性權重，加權求和得到整段手勢的代表向量，再送入分類器。

Day 1 從零推導 Transformer 的目的，正是為了理解這個機制的數學本質——為什麼 Attention 能讓模型自動聚焦在手勢動作最關鍵的幾幀，而不是平等對待所有幀。有了這個理解，才能在 Day 3 設計模型時清楚知道 Attention Pooling 在做什麼、為什麼選它。

```
Transformer Attention（論文）          BiLSTM Attention Pooling（本專案）
───────────────────────────────        ──────────────────────────────────
Q, K, V 計算相似度權重                  對每個時間步的 hidden state 計算權重
softmax 正規化                          softmax 正規化
加權求和 → context vector              加權求和 → 手勢代表向量 → 分類器
```

## 開發過程

整個專案在 **3 天**內完成，中間有一整天走錯方向、推倒重來。

### Day 1｜理論基礎

閱讀 *Attention Is All You Need*，手推 Scaled Dot-Product Attention、Multi-Head Attention、Positional Encoding、Feed-Forward Network 等所有核心公式，以 NumPy 從零實作 Transformer Encoder，與 PyTorch 官方實作數值比對（最大誤差 4.44e-16）。

這一天沒有寫任何手語相關的程式，但奠定了後續模型設計的理論基礎。

### Day 2｜第一次嘗試：網路爬蟲 × 失敗的資料來源

最初的構想是：台灣教育部的台灣手語辭典有 3,495 個詞彙的示範影片，如果能全部爬下來，就有現成的大型訓練資料集。

**實作過程：**
- 分析手語辭典網站的 API 結構（SPA，需用 Playwright 攔截請求）
- 以 requests 直接呼叫 API，下載全部詞彙的影片與語言學結構化資料
- 用 MediaPipe VIDEO 串流模式從 3,495 部影片提取 160,157 筆關鍵點特徵
- 訓練 MLP → BiLSTM，觀察指標

**遇到的問題：**

1. **畫質限制**：辭典影片為早期拍攝，解析度低、光線不足，MediaPipe 靜態模式僅偵測到 353 筆有效幀（偵測率不到 0.01%）。改用 VIDEO 串流模式雖大幅提升至 160,157 筆，但偵測失敗的幀仍以零向量填補，造成大量雜訊。

2. **類別爆炸**：3,495 個詞彙、平均每類僅 1.4 個訓練序列，模型根本沒有足夠的樣本學習。訓練集準確率 100%，驗證集僅 37%——典型的記憶而非學習。

3. **Domain Gap（最根本的問題）**：辭典影片由專業手語老師在攝影棚、固定角度、標準姿勢拍攝，與自己坐在桌前對著筆電攝影機比手語的動作分布完全不同。模型在辭典影片上訓練得再好，即時辨識時信心度也只有 6–15%，完全無法實用。

**結論：** 外部公開資料集的分布與真實使用場景差距太大，唯一解法是自己錄製訓練資料。

### Day 3｜重新出發：自錄資料 × 特徵工程 × 部署

放棄爬蟲方案後，重新設計整個 pipeline。

**資料收集：**

在 Claude Code 建議下，以「盡量浮誇、動作誇張、讓手勢間差異明顯」的方式自錄 32 個詞彙，每詞錄製 15 次，並在每次錄製中刻意改變手的位置（偏左、偏右、偏高、偏低）、速度（快打、慢打）和距離（靠近鏡頭、遠離鏡頭）。目的是讓模型學到手勢的本質特徵，而不是學到特定錄影條件下的規律。

**詞彙設計：**

從教育部部編台灣手語教材第 1–3 冊選出 62 個初始詞彙，訓練後發現部分詞彙手形過於相似（例如「聾人」和「是」都涉及指向動作），持續混淆。最終縮減至 32 個區分度高的詞彙。

**特徵工程（179 維）：**

不只擷取手部關鍵點，額外設計多個輔助特徵解決具體問題：

| 特徵 | 解決的問題 |
|------|-----------|
| `rel_wrist`（相對肩寬正規化位置）| 消除使用者距攝影機遠近造成的座標差異 |
| `palm_dir`（掌心朝向向量）| 區分正面/背面/側面手勢 |
| 雙臂方向（肘位置 + 上臂/前臂向量）| 區分手臂抬高/放下的動作 |
| 手臉距離（手腕→鼻/嘴/耳）| 區分「爸爸」「媽媽」「謝謝」等貼臉手勢 |
| 身體朝向（肩膀向量 cos/sin）| 消除使用者身體轉向造成的座標偏移 |
| 手指伸展分數（tip/mcp 距離比）| 量化每根手指是否伸直，直接編碼手形資訊 |

**解決過擬合：**

初版以 STRIDE=4 的滑動窗口切分序列，造成相鄰序列有 87.5% 的幀重疊——等同於把同一段影片給模型看了十幾遍，模型直接記住答案。改為 STRIDE=16 後，過擬合消失，驗證準確率穩定在 **95.31%**。

**EndPose Hybrid（即時辨識優化）：**

即時辨識時，手部動作中途的信心度往往跳動，容易誤判。觀察到每個詞彙都有一個「結束姿態」——手停下來的那一幀最能代表整個詞彙。因此在偵測到手腕靜止時，同時以兩種方式推論並加權合併：

```
最終分數 = 完整序列推論 × 0.6 + 靜止幀重複推論 × 0.4
```

這讓靜態手勢（如「嗎」「是」等）的信心度顯著提升，減少誤判。

**Web 部署：**

以 Flask + SocketIO 建立後端，瀏覽器每 100ms 送一幀 base64 JPEG，伺服器提取特徵並即時回傳 Top-3 結果。前端以純 HTML/CSS/JS 實作，無需任何前端框架。

## 技術細節

### 特徵工程（179 維）

| 區塊 | 維度 | 說明 |
|------|------|------|
| 雙手關鍵點 | 0–135 | 21 landmarks × 3 + rel_wrist(3) + palm_dir(2)，各手 68 維 |
| 身體錨點 | 136–145 | 鼻、肩、髖位置 |
| 雙臂方向 | 149–160 | 肘位置 + 上臂 / 前臂方向向量 |
| 手臉距離 | 161–166 | 手腕到鼻 / 嘴 / 耳的距離（÷ 肩寬正規化） |
| 身體朝向 | 167–168 | 肩膀向量 cos/sin（runtime 推導，無需重錄） |
| 手指伸展分數 | 169–178 | tip/mcp 距離比 ×10（runtime 推導，無需重錄） |

### 模型

- **架構**：BiLSTM（hidden=64）+ Attention Pooling → Linear(32 類)
- **訓練資料**：32 詞彙 × 15 次自錄，多位置、多速度
- **驗證準確率**：95.31%
- **序列長度**：32 幀（STRIDE=16 滑動窗口 + 高斯雜訊增強 + random crop）

### 特徵權重設計

```python
W_HANDSHAPE = 6.0   # 手指伸展分數（手形，最高優先）
W_FINGERTIP = 4.0   # 指尖座標
W_FINGER    = 3.0   # 其他手部關節
W_PALM_DIR  = 3.0   # 掌心朝向
W_FACE      = 3.0   # 手腕到臉距離
W_POSITION  = 2.5   # 手腕相對身體位置
W_ARM       = 2.5   # 手臂方向
W_ORIENT    = 2.0   # 身體朝向
W_ANCHOR    = 1.5   # 身體錨點
```
